# Sandbox Cross-Thread Collaboration Protocol v1.0

Status: Frozen / Mandatory\
Scope: Phase II and beyond\
Applies to: All Engine Reconstruction branches

------------------------------------------------------------------------

# I. Purpose

This protocol defines the mandatory information exchange format between:

1.  Engine Thread (Production Layer)
2.  Audit Thread (Governance & Structural Review Layer)

The objective is to:

-   Prevent context overflow
-   Preserve orthogonality
-   Avoid parameter drift
-   Maintain Archetype alignment
-   Reduce cross-thread noise

This protocol is binding once frozen.

------------------------------------------------------------------------

# II. Thread Role Separation

## A. Engine Thread (Production Layer)

Responsibilities:

-   Mathematical derivation
-   Engine design
-   Version iteration
-   Stability proof
-   Numerical method development

Prohibited actions:

-   Modifying Archetype definitions
-   Modifying Parameter Mapping
-   Introducing new personality dimensions
-   Reinterpreting frozen parameters

Engine thread must NOT output full derivations for audit.

Only structured summaries are allowed.

------------------------------------------------------------------------

## B. Audit Thread (Governance Layer)

Responsibilities:

-   Orthogonality verification
-   Parameter boundary enforcement
-   Cross-dimension contamination check
-   Governance compliance validation

Prohibited actions:

-   Re-deriving full engine math
-   Creating alternate engine designs
-   Expanding implementation details

Audit reviews structure, not implementation math.

------------------------------------------------------------------------

# III. Mandatory Engine Output Format

Every Engine revision must use the following structure:

------------------------------------------------------------------------

Engine Version: vX.Y

Modified Layer: (State / Cohesion / Movement / Target / Utility)

Affected Parameters: (List of personality parameters involved)

New Variables Introduced: (None / List)

Cross-Dimension Coupling: Yes / No (If Yes → specify)

Mapping Impact: None / Minor / Major (If not None → stop and escalate)

Governance Impact: None / Minor / Major (If not None → stop and
escalate)

Stability Property: Stable / Asymptotically Stable / Neutral / Unstable

Backward Compatible: Yes / No

Summary (≤ 15 lines): (Structural description only. No long formulas.)

------------------------------------------------------------------------

Full derivations remain in Engine thread only.

------------------------------------------------------------------------

# IV. Forbidden Transfer Rule

The following must NOT be copied into Audit Thread:

-   Full PDE expansions
-   Multi-page derivations
-   Long formula chains
-   Repeated definitions
-   Full code blocks

Only structural summaries are allowed.

------------------------------------------------------------------------

# V. Version Escalation Rule

If any of the following occur:

-   New personality parameter required
-   Parameter reinterpretation required
-   Mapping change required
-   Governance conflict detected

Engine thread must halt.

Version increment required:

Mapping Canonical vX+1 Governance Canonical vX+1

No silent modification permitted.

------------------------------------------------------------------------

# VI. Context Preservation Strategy

To prevent LLM context overload:

-   Audit Thread remains compact
-   Engine Thread contains full math
-   Only structural signatures travel between threads
-   No repeated canonical content

This maintains clean logical layering:

Archetype (Frozen) ↓ Mapping (Frozen) ↓ Engine (Iterative) ↓ Audit
(Structural Check)

------------------------------------------------------------------------

# VII. Activation Declaration

This protocol becomes active immediately upon freeze.

All future Phase II and Phase III work must comply.

Non-compliance invalidates structural integrity.
