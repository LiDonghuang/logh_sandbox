# 2D Phase Closeout and 3D Transition Memo (2026-03-26)

Status: Governance Memo - Repo-Side Fixed Record  
Authority: Governance Main Thread  
Implementation: Not authorized by this document

## 1. Purpose

This memo records one bounded governance judgment:

the current 2D mainline is ready for functional closeout, and the project may now transition from 2D contract cleanup into 3D baseline framing.

It does **not** authorize:

- new 2D implementation work
- broad `v4c` redesign
- immediate 3D runtime coding
- baseline replacement

## 2. Overall Judgment

The current 2D mainline is functionally closed.

Meaning:

- the phase's primary problem set has been sufficiently resolved or reframed
- first bounded rebuild success has been achieved
- key upstream layering is now clear enough to carry forward
- remaining issues have been downgraded to named residual debt

Meaningfully, this does **not** claim that:

- 2D is perfect
- every visual residual is gone
- current geometry is at a final aesthetic state

`v4a` established that removing the old shaping family is valuable:

- broad projection burden fell
- inward compaction fell
- the old polluted path is no longer the right place for continued interaction DOE

### B. No-enemy hidden fallback was removed

The no-enemy enemy-derived fallback no longer turns into inward self-centering.

This matters because it:

- removed a hidden inward masking source
- exposed the actual neutral-transit residuals more honestly
- reinforced that the main remaining issue was the restoration object

The accepted very small implementation proved that, in the bounded fixture path:

- expected-position restoration is better than centroid pull
- long-distance front stretching falls sharply
- broad projection burden does not rebound
- final radius ratio does not collapse inward again

### D. Upstream contract is now real

The 2D line now carries portable upstream contract assets:

- `ObjectiveLocationSpec v0.1`
- `Formation Specs v0.1`
- explicit separation between formation reference mapping and final legality/projection

## 5. Why Closeout Is Correct Now

Closeout is appropriate for four reasons.

### A. The main problem has closed

The phase started with three headline issues:

- wrong restoration object
- old shaping family polluting the neutral path
- unclear layering between objective, formation, and legality

All three now have bounded, positive resolution.

### B. Residuals have been downgraded

`R1` and `R2` are still real, but they are no longer headline blockers.

### C. The 2D payoff curve has changed

Further 2D work would now likely drift toward:

- profile micro-tuning
- stop/arrival micro-tuning
- visual polishing

That is no longer the highest-value frontier.

### D. Transition prerequisites are already present

The key carry-forward ingredients for 3D are already in place:

- objective ownership language
- formation reference language
- mapping vs legality layering
- a successful bounded rebuild example
