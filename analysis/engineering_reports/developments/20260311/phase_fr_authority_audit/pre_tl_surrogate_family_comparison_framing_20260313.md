# Pre-TL Surrogate Family Comparison Framing

Date: 2026-03-13
Status: Engineering comparison framing note
Scope: Structure how surrogate families should be compared as pre-TL substrate candidates without silently defining future `TL` behavior.

## 1. Purpose

This note does **not** rank permanent winners.

Its purpose is to classify surrogate families by boundary role:

- acceptable neutral substrate candidates
- sharp lower-bound references
- overly global upper references
- options that risk crossing into future `TL` semantics

## 2. Comparison principle

The comparison question is not:

- "Which surrogate is the true future targeting doctrine?"

The comparison question is:

- "Which surrogate families are acceptable low-level pre-TL substrate candidates?"

That means the evaluation priority is:

1. neutrality
2. smoothness / continuity
3. mirrored-diagnostic symmetry safety
4. locality / contact relevance
5. future extensibility for `TL`

and **not**:

1. doctrinal correctness
2. tactical cleverness
3. target-value intelligence

## 2.1 Refined comparison criteria

To keep the current comparison bounded and pre-TL, each surrogate family should now be read through four explicit criteria:

### A. Symmetry safety

- does the surrogate remain reasonably neutral under mirrored same-force diagnostic setups?
- does it avoid amplifying tiny candidate asymmetries into early fleet-level direction divergence?

### B. Smoothness

- does the surrogate vary continuously when the local enemy geometry changes slightly?
- is it robust against very thin nearest-candidate margins?

### C. Locality / contact relevance

- does it preserve short-horizon local contact structure?
- does it remain meaningfully tied to nearby contactable enemy geometry?

### D. Extensibility for future `TL`

- can future `TL` later write horizon or preference semantics onto it without finding those semantics already pre-encoded?
- does the surrogate remain a substrate rather than a hidden doctrine?

These criteria should dominate future bounded comparisons more than raw battle outcome.

## 3. Family classes

### 3.1 Sharp lower-bound references

Examples:

- raw `nearest-enemy`
- any single-point nearest-unit selector

Role in comparison:

- useful as sharp-end baseline
- useful as asymmetry-amplifying endpoint

Current engineering reading:

- acceptable as a comparison endpoint
- not currently preferred as neutral pre-TL substrate

### 3.2 Local smooth substrate candidates

Examples:

- nearest-k centroid
- local weighted centroid
- local cluster reference
- contactability-weighted local direction field

Role in comparison:

- currently the most promising candidate region
- likely to preserve local contact character while avoiding single-point instability

Current engineering reading:

- best current class to compare further under bounded audit
- but still subject to the four criteria above; not every local-smooth family should be assumed acceptable by default

### 3.3 Global smooth upper references

Examples:

- pure enemy centroid

Role in comparison:

- useful as symmetry-clean reference
- useful as upper smoothness benchmark

Current engineering reading:

- good as a comparison ceiling
- not automatically acceptable as the default doctrinal answer

### 3.4 Future-TL semantic candidates

Examples:

- explicit weak-point preference
- main-cluster doctrinal preference
- high-value target emphasis
- tempo-node preference
- fragility/value weighting

Role in comparison:

- outside current pre-TL substrate scope

Current engineering reading:

- not acceptable as current low-level substrate candidates
- belong to future `TL` or higher mechanism semantics

## 4. Current framing for specific families

### `nearest-enemy`

Current framing:

- keep as comparison endpoint
- sharp lower-bound reference
- symmetry-fragile, continuity-poor

### `nearest-k centroid`

Current framing:

- acceptable local smooth candidate family
- especially promising because it preserves locality while reducing single-point instability
- current bounded comparison suggests `nearest-5` is the strongest tested member of this family so far
- this is still a bounded comparative result, not a permanent winner declaration

### `local weighted centroid`

Current framing:

- acceptable local smooth candidate family
- especially aligned with the "contactability weight map" direction
- current bounded comparison suggests this family can achieve excellent symmetry and smoothness
- however, the tested weighted-local variant sits very close to pure centroid behavior, so locality dilution must be watched carefully

### `local cluster reference`

Current framing:

- acceptable if cluster logic remains low-semantic and physically grounded
- must not silently smuggle in doctrinal target-value semantics
- should now be treated with stricter scrutiny than plain `nearest-k centroid` or plain local weighted centroid

Specific scrutiny questions:

- is the cluster logic still just smoothing contactable local geometry?
- or is it starting to imply "which enemy group matters more"?
- is it weighting contactability, or silently choosing significance?

Current bounded comparison reading:

- still acceptable for continued comparison
- but currently less smooth than `nearest-5` and less neutral than the tested weighted-local variant
- therefore still a higher-scrutiny family, not a preferred default candidate

### `pure enemy centroid`

Current framing:

- symmetry-clean upper reference
- useful comparator
- not automatically preferred as permanent substrate

## 5. What counts as acceptable low-level grounding

Acceptable grounding for current surrogate-family comparison:

- current geometry
- locality
- short-horizon contactability
- short-horizon pressure feasibility
- range proximity
- light spacing feasibility

Unacceptable grounding at this stage:

- target value judgment
- strategic importance
- explicit preference for main cluster in doctrinal sense
- explicit weak-point exploitation semantics
- parameter-specific personality preference injection

## 6. What should be reported in future comparisons

Future bounded surrogate comparisons should report at least:

- mirrored target-direction symmetry behavior
- early orientation-field skew
- contact-entry timing effects
- engaged-split timing
- movement-phase mirror-error behavior
- whether the surrogate stays continuous when nearest-candidate margins are thin
- whether the surrogate remains locally contact-relevant without drifting too global
- whether the surrogate behaves like:
  - a sharp lower-bound endpoint
  - a neutral local candidate
  - an overly global reference
  - or a semantic overreach into future `TL`

Where possible, the current four criteria should be read through a bounded trade-off matrix:

- stronger symmetry safety
- stronger smoothness
- preserved locality/contact relevance
- preserved extensibility for future `TL`

This is important because one family may improve one criterion by collapsing toward another extreme:

- too sharp
- too global
- or too semantic

## 7. Required interpretive discipline

No surrogate family should be described as:

- "the true TL answer"
- "the correct targeting doctrine"
- "the right battlefield intelligence rule"

At this stage, families may only be described as:

- substrate candidates
- boundary references
- or semantic overreach risks

## 8. Bottom line

Current comparison framing should remain:

- `nearest-enemy` = sharp lower-bound endpoint
- local smooth / contactability-based families = main candidate region
- pure enemy centroid = symmetry-clean upper reference
- semantic preference families = future `TL` territory

Current bounded comparison does **not** yet authorize a permanent winner.

The present engineering reading is only:

- `nearest-5` is currently the strongest tested balance candidate among explicitly local smooth families
- `weighted_local` is currently the strongest symmetry/smoothness candidate, but at risk of becoming too global
- `local_cluster` remains admissible but under stricter scrutiny

These are trade-off observations, not doctrinal decisions.

This keeps current work inside pre-TL substrate audit and prevents silent drift into premature `TL` design.
