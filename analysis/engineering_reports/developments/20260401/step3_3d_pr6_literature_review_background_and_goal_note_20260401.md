Status: discussion-support note only  
Scope: background and target-setting for Human + Governance literature discussion  
Authority: engineering support note, not implementation approval, not doctrine activation

## I. Purpose

This note is meant to help Governance do a more useful next step than simple agreement.

The immediate purpose is twofold:

1. explain why the current PR `#6` transition carrier has now reached a real methodological boundary, while still having completed an important historical role
2. state the current Human + Engineering consensus about what the **next-generation formation-transition carrier** likely needs to become

This note is therefore not a "please bless our current direction" memo.

It is a request for Governance to:

- actively explore the literature space
- compare alternative conceptual routes
- challenge weak assumptions
- and push back where the current Human + Engineering consensus may still be too local, too convenient, or too repo-biased

In short:

- do not merely echo the current line
- test it
- sharpen it
- and, where needed, resist it

## II. What PR #6 Has Already Accomplished

PR `#6` should not be read as disposable just because the current carrier hit a boundary.

It has already produced bounded value that remains important:

- spacing split legitimacy
- restore-line legitimacy
- battle-side restore visibility
- soft-carrier experiments
- human-readable evidence line
- explicit failure learning about reference-only carrier limits
- opening of transition-movement seams
- first bounded battle standoff `d*`
- first bounded unit-level attack-direction-aware speed envelope

So the correct read is not:

- "the whole line was wrong"

The correct read is:

- the branch successfully performed a historical role
- but the **current carrier form** has now reached the edge of what it can honestly carry

That distinction matters.

## III. What Historical Role the Current Carrier Has Finished

The current carrier helped establish several things that were previously still uncertain:

1. formation cannot be treated as only a fixture-only restore illusion
2. reference / restore semantics do matter in battle-relevant paths
3. spacing split is real and worth preserving
4. transition must be read as a coupled problem, not as "define better reference and let runtime follow"

In that sense, the current carrier has already done one major job:

- it proved that the older "reference-only" reading was too weak

Then, by failing in more specific ways, it did a second major job:

- it exposed what the next problem really is

That next problem is not simply:

- "tune restore harder"
- "pick a better rectangle"
- "pick a better band split"

It is:

- how to own **formation transition** honestly

## IV. Why the Current Carrier Has Reached a Methodological Boundary

The current carrier still leans too much on an approach of the following kind:

- define a relatively explicit target body
- define a gradual path toward it
- then pull units toward that body

This is already better than rigid slot restore.
But it is still too close to:

- geometry-first transition

That becomes especially visible in mismatch cases such as:

- `1 -> 4`
- `2 -> 4`
- `2 -> 1`
- `4 -> 1`

The current Human read has made the following especially clear:

### 1. `1 -> 4` is misleading if read alone

`1 -> 4` can sometimes look relatively orderly.
But this may contain symmetry luck:

- splitting into two wings
- moving them laterally
- and getting something that superficially resembles the target

This is not strong evidence that a generic transition carrier is already working.

### 2. `2 -> 4` exposes the weakness much more honestly

`2 -> 4` looks noticeably more awkward and less stable.
That matters because it suggests the current carrier is still highly sensitive to initial arrangement geometry.

So the branch should no longer let `1 -> 4` function as a comfort case.

### 3. Target morphology and actual unit body still diverge too often

Current internal diagnostics already suggest cases where:

- the carrier's current target extents converge
- but the live-unit body does not honestly follow

That means the remaining gap is not just "wrong target values."
It is also a problem of:

- transport continuity
- movement realization authority

### 4. Terminal regularization remains suspicious

A repeated Human read is:

- arrival can still trigger a sudden hardening / forced regularization

That is another sign that the current method still tends to "resolve into a shape"
instead of continuously becoming the right body in motion.

## V. Why the Current Extent Semantics Should Be Treated as Temporary

Current active diagnostics rely heavily on:

- `forward_extent`
- `lateral_extent`
- half-extents such as `fcur`, `afwd`, `lcur`, `alat`

These are useful **temporary observer/carrier semantics**.
They are helping Engineering and Human measure what is happening.

But they should not be mistaken for a mature formation ontology.

Reasons:

1. even for rectangles, the read is already somewhat suspicious
2. for non-rectangular targets, this representation is obviously insufficient
3. for future 3D formation ownership, simple forward/lateral half-extents will be even less adequate

So these quantities should currently be read as:

- bounded diagnostic semantics
- maybe first-order carrier quantities
- not a final answer to "what a formation is"

This concern should be stated clearly in the Governance discussion.

If Governance later accepts a literature direction that makes these quantities less central,
that should not be treated as a contradiction.
It would be a natural maturation.

## VI. Current Human + Engineering Consensus About the Next Carrier

Current shared read:

- the next carrier should not begin by telling the system the exact sharp target geometry
- it should first own a softer, more continuous target body
- then allow later refinement

The shortest expression of this read is:

- **soft body first, refinement later**

This is not yet a final mechanism design.
It is a methodological direction.

The key intuition is:

- a fleet should first become approximately the right kind of body
- only later should it be asked to look like a more sharply defined shape

This should be read as a candidate response to the current failures:

- odd/even-style discrete splitting luck
- shell-like external success with disordered interior
- transition results that depend too much on special rectangular symmetry
- terminal "snapping into shape"

---

## VII. What the Next-Generation Carrier Likely Needs to Own

Current Human + Engineering consensus is that the next carrier probably needs to own at least four things more honestly:

### 1. A continuous soft morphology body

Not just:

- a target rectangle
- a target slot field
- a target band lattice

But some softer continuous morphology read, likely including a small set of low-order quantities such as:

- main axis
- body extent
- center/wing differential
- possibly front curvature or front mass distribution

### 2. Topology continuity through transition

Units should not merely be dragged toward a target shell.

The carrier must have a better answer for:

- how body membership deforms
- how continuity is preserved
- how internal order avoids collapsing into chaos

### 3. Movement realization that is genuinely transition-aware

This includes at least:

- turning
- slowing
- hold / arrival behavior
- transition-aware movement authority

The branch has already started opening this layer.
But it is not yet enough.

### 4. Separation between coarse shaping and later refinement

This may be the most important methodological point.

The carrier likely needs to distinguish:

- first becoming the right broad body
- then refining toward a crisper shape if and when that becomes appropriate

## VIII. What the Literature Review Should Explore

The literature discussion should stay at **method-family level first**.

The first useful question is not:

- "which exact paper should we copy?"

The first useful question is:

- which kinds of methods match the problem we now understand?

The most relevant families currently look like:

### A. Continuous formation morphing

Methods where a group transitions between shapes through a continuous intermediate body,
instead of direct discrete reassignment.

### B. Crowd / multi-agent formation control with topology continuity

Methods where agents preserve continuity or local ordering during shape change,
rather than simply re-snapping into a new pattern.

### C. Continuum / deformable-body control

Methods that treat the group more like a deforming body than a set of units chasing a finished template.

### D. Potential-field / density-field / implicit-shape methods

Methods that define desired massing, occupancy, or body pressure indirectly,
possibly allowing a softer first phase of shape emergence.

### E. Coarse-shape-first, refinement-later methods

Any method family that explicitly distinguishes:

- broad body convergence
- later detailed alignment / refinement

### F. Military formation transition / maneuver doctrine

Not as code to copy,
but as conceptual guidance about:

- how real formations change shape
- what "orderly" transition means
- how slowing / turning / interval control / frontage change should be understood

This last category is especially important because the Human read is strongly doctrine-sensitive,
not only mathematically sensitive.

## IX. What Governance Should Actively Test, Not Merely Accept

This section is intentionally direct.

Governance is asked to test and push back on the following possible weak spots in the current Human + Engineering consensus:

### 1. Is "soft body first, refinement later" really the right central split?

It seems promising.
But Governance should still ask:

- is this truly the right abstraction,
or just the next attractive intuition?

### 2. Are current low-order morphology quantities enough?

Current candidate quantities such as:

- axis
- forward extent
- lateral extent
- center/wing differential

may be helpful, but Governance should ask:

- are these genuinely meaningful,
or are they still too tied to rectangle thinking?

### 3. Is the repo over-valuing smooth appearance over structural honesty?

Some visually smoother transitions may still be methodologically wrong.
Governance should challenge any proposal that merely looks cleaner without owning the right semantics.

### 4. Is the repo underestimating how much true transition needs movement realization?

Current work has already shown that target morphology alone is insufficient.
Governance should actively test whether the next carrier needs a deeper movement layer earlier than Engineering would prefer.

### 5. Is there a better decomposition than the current Human + Engineering one?

If Governance sees a better long-term split than:

- soft body first
- refinement later

it should say so explicitly.

This is the place for constructive pushback.

## XI. Bottom Line

PR `#6` has not failed in a disposable way.
It has successfully done part of its historical work.

But the current carrier has reached a real methodological boundary:

- too much target-shape-first thinking remains
- transition behavior still depends too much on special geometry
- current extent semantics are useful but temporary
- the method is not yet a generic formation-transition carrier

The next discussion with Governance should therefore pursue two goals in order:

1. explain clearly why the current carrier has reached that boundary
2. explore, with real pushback, what a better next-generation carrier should be

Current Human + Engineering consensus is:

- soft body first
- refinement later

But Governance should treat that as a serious candidate to test,
not as a conclusion to endorse automatically.
